PROJECT SUMMARY
================================================================================

THE PROBLEM:
------------
Logo coordinates stored 50px smaller than visual position due to CSS padding
not being compensated in coordinate calculations.

THE SOLUTION:
-------------
- JavaScript: Add 50px offset when saving, subtract when loading
- PHP: Subtract 50px offset when generating print API payloads
- Metadata: Use offset_applied flag for backward compatibility

CODE CHANGES:
-------------
- JavaScript: 38 lines added, 13 OFFSET-FIX markers
  File: public/js/dist/designer.bundle.js
  
- PHP: 36 lines added, 5 OFFSET-FIX markers
  File: includes/class-octo-print-api-integration.php

- Total: 74 lines across 2 files

BACKWARD COMPATIBILITY:
-----------------------
100% GUARANTEED via metadata.offset_applied flag
- Old designs: Use coordinates as-is (no metadata)
- New designs: Subtract offset (metadata.offset_applied = true)
